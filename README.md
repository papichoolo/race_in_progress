- **AI drive (press L): purple car follows a baked racing line, lap after lap.** Two-stage pipeline. **Offline (`scripts/extract-racing-line.mjs`):** walks each GLB, rasterises the per-map `racingLineMaterials` to a binary mask (Spa 3 m/cell, Nordschleife 2 m/cell, Suzuka 1.5 m/cell), morphological close to bridge mesh micro-gaps, distance-transform → "core" mask (cells ≥ coreFrac × maxDT) which carves off pit-lane stubs and paddock fragments, picks the connected component containing the spawn, does a geodesic BFS from the spawn to find the antipode (~halfway around the loop), then a BFS from the antipode — cells where `d1 + d2 ≈ d1[antipode]` form a thin ring that traces the FULL lap (the geodesic-circle trick avoids the previous nearest-neighbour tour that kept boxing itself in at junctions). Zhang-Suen thins the ring; Hierholzer Euler circuit walks the resulting graph; output is resampled to uniform arclength (8 m Nordschleife, 6 m GP/Suzuka, 10 m Spa) and run through the existing min-curvature optimiser + forward/backward speed sweep. Outputs `public/racing-lines/<id>.json` — Nordschleife 1834 wpts, GP 3722, Spa 5453, Suzuka 2846 — each waypoint `{x, z, v}` with `v` clamped to `min(V_CAP, sqrt(μ·g/κ))`. **Runtime (added to `src/main.js`):** `loadRacingLine()` fetches the JSON in `loadTrack()`; `L` key toggles `ai.enabled`; `aiUpdate(dt)` runs immediately after `updateInput(delta)` so it overrides keyboard/gamepad/touch every frame. Controller is kinematic-bicycle pure-pursuit: incremental closest-point search (scan 80 waypoints forward, 6 back), speed-adaptive lookahead `5 + speed × 0.45` clamped to 6–28 m, target waypoint angle in car-local frame, `δ = atan2(2·L·sin(α), Ld)` with L = 2.6 m wheelbase, mapped to `input.steer ∈ [-1, +1]` over a 35° max. Speed PID: target = 7-waypoint smoothed average ahead, throttle ramps as `0.3 + dv × 0.12` when below target, brake as `(-dv - 1.5) × 0.08` when ≥ 1.5 m/s over; coast at 0.15 throttle in the dead-zone; trail-brake bonus while turning (0.25–0.85 steer); reverse safety = full brake if forward-velocity dot < -2. Any deliberate human input (WASD / arrows / space / handbrake / touch / gamepad steer > 0.05) immediately cancels AI mode. **Visuals:** car body materials swap to bright purple (0x9933FF, skipping pure-black trim like windows/tyres + brake-red), restored on toggle off. A 0.6 m Catmull-Rom TubeGeometry traces the racing line above the track at +0.4 m (raycast straight down at each waypoint against the chunked track mesh so the ribbon follows Nordschleife's elevation changes instead of clipping into hills); transparent 0.78 magenta, `depthWrite:false`, `renderOrder:999` so it reads cleanly through chase-cam. Top-centre "AI DRIVE" badge with letter-spacing 2 px on a translucent purple chip. **(HEAD)**

- Multiplayer overhaul: spawn slots, collision physics, minimap dots. Three long-standing MP bugs fixed by a dedicated multiplayer-systems subagent. **Spawn slots**: replaced the racy `setTimeout(() => _applySpawnSlot(room.peers().length), 300)` (which let two simultaneous joiners claim the same slot) with deterministic sorted-peer-ID slot assignment in `_computeLocalSlot()` — every client sorts `{ selfId, ...peerIds }` alphabetically and uses its own index. Same answer on every machine, no coordination needed. `_reapplyLocalSlotIfSafe()` re-shuffles all peers on every `peerJoin` / `peerLeave` during lobby + ready_check (never mid-race), and `_applySpawnSlot` now honours slot 0 (was previously early-returning, so a peer whose slot recomputed from non-zero back to 0 wouldn't actually teleport home). **Collision-launch fix**: three compounding causes addressed. (1) `RemoteCar._buildPhysics` now scales the cuboid collider by the current map's `carScale` to match the 1.5× visual on Spa/Suzuka — without this, the visual mesh was visibly overlapping before physics noticed contact, and the penetration recovery launched the local chassis into the skybox. New `rebuildPhysicsForMap()` re-creates the collider on map swap. (2) `RemoteCar.update` clamps per-frame kinematic delta to 2 m (~120 m/s @60 fps); larger jumps use `setTranslation` directly instead of `setNextKinematicTranslation`, which avoids Rapier synthesising a phantom 150+ m/s velocity from snap-gap teleports. (3) Local chassis collider was being created with `chassisFriction` (0.9–1.0) passed as its restitution by the shared `physics.addMesh` helper, so every car-car contact was being amplified to MAX(0.9, 0.0) = 0.9 bounce. Now caches the collider on a module-scoped `chassisCollider`, sets `friction = chassisFriction` and `restitution = 0` explicitly, re-applied on every car swap in `applyCarConfig`. Remote colliders also pinned to `friction 0.8 / restitution 0` so cars scrape, never bounce. **Minimap dots**: each remote peer now gets a small DOM circle in `minimap.containerEl`, coloured by `_colorForPeer(peerId)`. `_updateMinimapPeerDots()` runs every frame inside `renderMinimap()`, projecting remote car positions into the ortho-camera screen basis (±250 m frustum → ±100 px canvas); off-screen peers clamp to the rectangular boundary at 0.6 opacity in their actual direction. Dots hide when no snaps received yet, and the minimap container's `display:none` handles the toggle-off case. All non-overlapping with the single-player code path — `multiplayer.room === null` short-circuits the helpers. **(HEAD)**

- Traction control: per-car TC + relaxed defaults so TC ON doesn't cripple cars. Shipped TC config was strangling acceleration on every car (5+ seconds to pick up from a standstill above 16 km/h) because `minMult: 0.15` floored throttle at 15% the moment slip exceeded `0.13`. That's "limp mode," not TC. Defaults bumped to `slipThreshold: 0.16, cutGain: 4.0, minMult: 0.45`. Six cars got per-car `tc: { slipThreshold, minMult, cutGain }` overrides — F1 `{0.28/0.75/2.5}` because real F1 TC was nearly imperceptible, God Car `{0.32/0.80/2.0}` because the fantasy car shouldn't ever feel restricted, Rally Turbo `{0.24/0.60/3.0}` for AWD launches, etc.

- DualSense adaptive triggers via WebHID. Self-contained protocol layer (Sony VID `0x054C`, USB report `0x02` 47 B / Bluetooth report `0x31` 78 B with CRC32 trailer), four mode helpers (`Off / Feedback / Weapon / Vibration`), ~30 Hz flush throttle with idempotent state-diff, 3-strike auto-disable on write errors, silent reconnect via `navigator.hid.getDevices()` + `dualsenseAuthorized` localStorage flag. Permission UX: a small `+ adaptive triggers` button in the controls panel that only renders when a DualSense-looking pad is connected; click triggers the device picker, success swaps it to `✓ triggers on`. Effects: **R2 throttle** = Feedback force 1→6 ramped by RPM/redline, flips to Weapon clack at the rev limiter, ~200 ms force-4 bump on TC engage. **L2 brake** = Feedback force 0–8 by brake input + a trail-braking bonus (up to +2 force scaled by `|steer| × brake`) so the pedal stiffens when you brake mid-corner — the moment a real driver wants the most pedal feel. ABS engagement overrides to a 10 Hz vibration to mimic pedal pulsing. Reverse-engaged shows a constant force-1 reminder. All effects scaled by the existing **RUM** master slider; `RUM = 0` flushes both triggers OFF. Safari / Firefox / non-Sony pads = completely inert, zero console output. Existing standard-rumble path untouched.
