# race_in_progress

## Latest Commit

- **DualSense triggers: per-car force-feedback profiles.** Every car in the fleet (Hatchback, Muscle V8, Sport Flat-six, Rally Turbo, Supercar V12, F1, God Car) now ships a `dsProfile` literal that customizes all 7 trigger cues — revLimit, shiftTick, wheelspin, slide, abs, kerb, brakePressure — to match the car's character. The mapper reads `currentCar.dsProfile.*` for every force range, frequency, startPos, hold-ms, cooldown-ms, and detection threshold. Highlights: Hatchback gets an older 8 Hz ABS pulse and a plastic 10 Hz limiter fizz; Muscle V8 has a 5 Hz pushrod-thud limiter and an early-firing 0.26 wheelspin threshold so burnouts are the signature cue; Sport Flat-six gets crisp 15 Hz electronic limiter + 911 rear-rotation slide cue at low 0.34 threshold + 16 kN kerb floor (stiff short-travel transmits everything); Rally Turbo's slide cue triggers early (0.32) and holds long (280 ms) for Scandi flicks, kerbs need 26 kN to register through long-travel suspension; Supercar V12's rev limiter is a deliberate 12 Hz "shoulder tap" with 300 ms hold (heavy crank inertia), carbon-ceramic ABS at 15 Hz; F1 sits at the protocol's freq ceiling for revLimit + ABS, near-seamless 35 ms paddle blip at startPos 200, spool-diff wheelspin at 0.22 (instant on/off), brutal kerbs at just 8 kN; God Car is the fantasy machine — 15 Hz screams, 30 ms micro-shifts, magic-tyre thresholds (0.50/0.52) so wheelspin/slide are "you broke physics" alarms. The force-amplitude rule stays load-bearing: all profiles obey momentary ≤ 130 / sustained ≤ 110 / brake ≤ 100 caps, differentiation lives in freq + startPos + threshold + duration. Designed by 7 parallel design agents (one per car) reading the actual physics params + mapper code. `DS_DEFAULT_PROFILE` fallback constant guards against a car shipping without a profile. **(HEAD)**

- DualSense triggers: gear-shift always wins, ambient cues stop spamming. Three-part fix: (1) promoted shift-tick above wheelspin/slide so a deliberate driver action beats ambient cues; (2) added 700 ms / 900 ms re-arm cooldowns on wheelspin/slide so slip-ratio bobbing can't refire every frame; (3) tightened thresholds (slip 0.22 → 0.35/0.40, speed gate 18 → 22 km/h, kerb 12 kN → 20 kN). Removed the `setR2`/`setL2` mode-hold wrapper entirely — it was the actual mechanism swallowing shift events. Also kept the `resetDualsenseState()` helper from the prior commit, called from R-reset / car-swap / map-swap paths.

- DualSense triggers: reset edge-state on R / car-swap / map-swap. New `resetDualsenseState()` helper wipes all per-frame edge trackers (`prevSusp`, `prevShiftCd`, `prevTcEngaged`, `prevLatSlip`, `prevAbsEngaged`) and clears every `*Until` cue timer, then flushes both triggers to OFF. Called from the R-reset path (`applyVehicleForces` reset block), the car-swap path (`changeCar`), and the map-swap teleport. Fixes the post-reset behaviour where the teleport's suspension-load and slip discontinuities synthesised false kerb / wheelspin / chassis-slide cues.

- DualSense adaptive triggers via WebHID. Self-contained protocol layer (Sony VID `0x054C`, USB report `0x02` 47 B / Bluetooth report `0x31` 78 B with CRC32 trailer), four mode helpers (`Off / Feedback / Weapon / Vibration`), ~30 Hz flush throttle with idempotent state-diff, 3-strike auto-disable on write errors, silent reconnect via `navigator.hid.getDevices()` + `dualsenseAuthorized` localStorage flag. Permission UX: a small `+ adaptive triggers` button in the controls panel that only renders when a DualSense-looking pad is connected; click triggers the device picker, success swaps it to `✓ triggers on`. Effects: **R2 throttle** = Feedback force 1→6 ramped by RPM/redline, flips to Weapon clack at the rev limiter, ~200 ms force-4 bump on TC engage. **L2 brake** = Feedback force 0–8 by brake input + a trail-braking bonus (up to +2 force scaled by `|steer| × brake`) so the pedal stiffens when you brake mid-corner — the moment a real driver wants the most pedal feel. ABS engagement overrides to a 10 Hz vibration to mimic pedal pulsing. Reverse-engaged shows a constant force-1 reminder. All effects scaled by the existing **RUM** master slider; `RUM = 0` flushes both triggers OFF. Safari / Firefox / non-Sony pads = completely inert, zero console output. Existing standard-rumble path untouched.
